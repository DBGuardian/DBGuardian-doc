## 快速创建脚本

### Windows PowerShell

```powershell
# 创建新的测试项目
$projectName = "dbguardian-test-java17-sb32-jpa-pg"

# 创建项目目录
New-Item -ItemType Directory -Path $projectName -Force

# 复制基础配置
Copy-Item -Path "dbguardian-test/pom.xml" -Destination "$projectName/pom.xml"
```

### Bash / Git Bash

```bash
# 创建新的测试项目
PROJECT_NAME="dbguardian-test-java17-sb32-jpa-pg"

# 创建项目目录
mkdir -p $PROJECT_NAME

# 复制基础配置
cp dbguardian-test/pom.xml $PROJECT_NAME/pom.xml
```

## 项目目录结构

每个测试项目应遵循以下结构：

```
dbguardian-test-{tech-stack}/
├── pom.xml                          # Maven 配置
├── src/
│   ├── main/
│   │   ├── java/com/dbguardian/test/
│   │   │   ├── {AppName}Application.java
│   │   │   ├── config/              # 配置类
│   │   │   ├── controller/          # 控制器
│   │   │   ├── service/            # 服务层
│   │   │   ├── mapper/             # Mapper/Repository
│   │   │   ├── entity/             # 实体类
│   │   │   └── model/              # DTO/VO
│   │   └── resources/
│   │       ├── application.yml      # 主配置
│   │       ├── application-dev.yml # 开发环境
│   │       └── application-prod.yml# 生产环境
│   └── test/
│       └── java/com/dbguardian/test/
│           ├── ReadWriteSplittingTest.java
│           ├── FailoverTest.java
│           └── LoadBalanceTest.java
└── README.md                        # 项目说明
```

## 测试用例清单

每个测试项目应包含以下核心测试用例：

| 测试类别 | 测试用例 | 说明 |
|---------|---------|------|
| **读写分离** | `testReadFromSlave` | 验证 SELECT 路由到从库 |
| **读写分离** | `testWriteToMaster` | 验证 INSERT/UPDATE/DELETE 路由到主库 |
| **读写分离** | `testTransactionReadWrite` | 验证事务内操作路由到主库 |
| **负载均衡** | `testSlaveLoadBalance` | 验证从库负载均衡策略 |
| **负载均衡** | `testMasterFailover` | 验证主库故障时的负载均衡 |
| **故障转移** | `testMasterFailover` | 验证主库故障自动切换 |
| **故障转移** | `testSlaveFailover` | 验证从库故障自动切换 |
| **故障转移** | `testMasterRecovery` | 验证故障主库恢复后重新加入 |
| **注解路由** | `testForceMasterAnnotation` | 验证 `@ForceMaster` 注解 |
| **注解路由** | `testReadOnlyAnnotation` | 验证 `@ReadOnly` 注解 |
