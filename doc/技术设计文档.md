## 2. 支持的集群架构

DBGuardian 支持多种数据库集群拓扑结构：

### 2.1 基础架构模式

| 模式 | 标识 | 架构类型 | 读能力 | 写能力 | 适用场景 |
|-----|------|---------|-------|-------|---------|
| **M1S1** | `SINGLE_MASTER` | 单主单从 | 1x | 1x | 开发/测试/小规模生产 |
| **M1Sn** | `SINGLE_MASTER_MULTI_SLAVE` | 单主多从 | nx | 1x | 读密集型应用 |
| **MnSn** | `MULTI_MASTER_MULTI_SLAVE` | 多主多从 | nx | mx | 高可用读写 |
| **MnS0** | `MULTI_MASTER_NO_SLAVE` | 多主无从 | 0x | mx | 分库写入 |
| **M2S2** | `DUAL_MASTER_DUAL_SLAVE` | 双主双从 | 2x | 2x | 对称高可用 |

### 2.2 高级架构模式

| 模式 | 标识 | 架构类型 | 读能力 | 写能力 | 适用场景 |
|-----|------|---------|-------|-------|---------|
| **Cascade** | `CASCADE_REPLICATION` | 级联复制 | nx | 1x | 减轻主库复制压力 |
| **Ring** | `RING_REPLICATION` | 环形复制 | nx | mx | 多数据中心部署 |
| **Star** | `STAR_REPLICATION` | 星型复制 | nx | 1x | 大规模从库部署 |
| **GR** | `GROUP_REPLICATION` | 集群复制 | nx | nx | 强一致性高可用 |
| **Galera** | `GALERA_CLUSTER` | Galera集群 | nx | nx | 同步多主复制 |
| **Sharding** | `SHARDING` | 分片集群 | nx | nx | 超大规模数据 |

详细架构设计请参考 [集群架构设计.md](./集群架构设计.md)

## 3. 核心类设计

### 3.1 ClusterDataSourceProperties（集群配置）

```java
/**
 * DBGuardian 集群配置属性
 * 支持单主单从、单主多从、多主多从等多种集群拓扑
 */
@ConfigurationProperties(prefix = "spring.datasource.dbguardian")
public class ClusterDataSourceProperties {

    /** 集群模式 */
    private ClusterMode clusterMode = ClusterMode.SINGLE_MASTER;

    /** 主库配置（支持多个） */
    private List<MasterConfig> masters = new ArrayList<>();

    /** 从库配置（支持多个） */
    private List<SlaveConfig> slaves = new ArrayList<>();

    /** 负载均衡配置 */
    private LoadBalanceConfig loadBalance = new LoadBalanceConfig();

    /** 故障转移配置 */
    private FailoverConfig failover = new FailoverConfig();

    /** 集群模式枚举 */
    public enum ClusterMode {
        SINGLE_MASTER,           // 单主单从
        SINGLE_MASTER_MULTI_SLAVE, // 单主多从
        MULTI_MASTER_MULTI_SLAVE,   // 多主多从
        MULTI_MASTER_NO_SLAVE,      // 多主无从
        DUAL_MASTER_DUAL_SLAVE      // 双主双从
    }
}

/**
 * 主库配置
 */
public static class MasterConfig {
    private String id;           // 唯一标识
    private String url;          // JDBC URL
    private String username;
    private String password;
    private int priority = 100;  // 优先级
    private String peerId;       // 故障转移目标
    private HikariConfig hikari;
}

/**
 * 从库配置
 */
public static class SlaveConfig {
    private String id;
    private String url;
    private String username;
    private String password;
    private int weight = 100;   // 负载权重
    private String masterRef;    // 所属主库ID
    private HikariConfig hikari;
}
```

### 3.2 DataSourceRegistry（数据源注册中心）

```java
/**
 * 数据源注册中心
 * 统一管理所有主库和从库的生命周期和可用性
 */
@Component
public class DataSourceRegistry {

    // 主库Map: id -> DataSourceWrapper
    private final Map<String, DataSourceWrapper> masters = new ConcurrentHashMap<>();

    // 从库Map: id -> DataSourceWrapper
    private final Map<String, DataSourceWrapper> slaves = new ConcurrentHashMap<>();

    // 可用主库ID集合
    private final Set<String> availableMasterIds = ConcurrentHashMap.newKeySet();

    // 可用从库ID集合
    private final Set<String> availableSlaveIds = ConcurrentHashMap.newKeySet();

    // 注册/注销方法
    public void registerMaster(String id, DataSourceWrapper wrapper);
    public void registerSlave(String id, DataSourceWrapper wrapper);

    // 获取可用数据源列表
    public List<DataSourceWrapper> getAvailableMasters();
    public List<DataSourceWrapper> getAvailableSlaves();

    // 获取最高优先级主库
    public Optional<DataSourceWrapper> getHighestPriorityMaster();
}
```

## 3. 数据流

### 3.1 正常读操作流程

```
Service.selectXxx()
       ↓
MyBatis Mapper (DataSourceAspect 拦截)
       ↓
RoutingDataSource.setMethodType(READ)
       ↓
HikariPool-Slave.getConnection()
       ↓
Slave DB 执行查询
```

### 3.2 正常写操作流程

```
Service.insertXxx()
       ↓
MyBatis Mapper (DataSourceAspect 拦截)
       ↓
RoutingDataSource.setMethodType(WRITE)
       ↓
HikariPool-Master.getConnection()
       ↓
Master DB 执行写入 + 复制到 Slave
```

### 3.3 故障转移流程

```
1. 健康检查发现 Master 不可用
          ↓
2. DatasourceCoordinationService.tryAcquireFailoverLock()
          ↓ (成功获取锁)
3. DataSourceConfig.handleFailover()
          ↓
4. 检查 Slave 数据一致性
          ↓
5. 更新 Replication 配置 (change master to)
          ↓
6. SlaveDataSource → 新主库
          ↓
7. coordinationService.broadcastStatusChange(SLAVE_PROMOTED)
          ↓
8. 其他实例收到通知，同步状态
          ↓
9. DatasourceStatusListener 更新本地状态
```

## 4. 配置示例

### 4.1 单主单从配置（向后兼容）

```yaml
spring:
  datasource:
    master:
      url: jdbc:mysql://master:3306/mydb
      username: root
      password: secret
    slave:
      url: jdbc:mysql://slave:3306/mydb
      username: root
      password: secret
```

### 4.2 单主多从配置（M1Sn）

```yaml
spring:
  datasource:
    dbguardian:
      enabled: true
      cluster-mode: single-master-multi-slave

      masters:
        - id: master-1
          url: jdbc:mysql://master:3306/mydb
          username: root
          password: secret
          priority: 100

      slaves:
        - id: slave-1
          url: jdbc:mysql://slave1:3306/mydb
          weight: 100
        - id: slave-2
          url: jdbc:mysql://slave2:3306/mydb
          weight: 100
        - id: slave-3
          url: jdbc:mysql://slave3:3306/mydb
          weight: 80

      load-balance:
        read-strategy: weighted-round-robin
```

### 4.3 多主多从配置（MnSn）

```yaml
spring:
  datasource:
    dbguardian:
      enabled: true
      cluster-mode: multi-master-multi-slave
      sharding-column: user_id

      masters:
        - id: master-0
          url: jdbc:mysql://master0:3306/mydb_0
          priority: 100
        - id: master-1
          url: jdbc:mysql://master1:3306/mydb_1
          priority: 100

      slaves:
        - id: slave-0-1
          url: jdbc:mysql://slave0:3306/mydb_0
          master-ref: master-0
          weight: 100
        - id: slave-1-1
          url: jdbc:mysql://slave1:3306/mydb_1
          master-ref: master-1
          weight: 100
```

### 4.4 双主双从配置（M2S2）

```yaml
spring:
  datasource:
    dbguardian:
      enabled: true
      cluster-mode: dual-master-dual-slave

      masters:
        - id: master-a
          url: jdbc:mysql://master-a:3306/mydb
          priority: 100
          peer: master-b
        - id: master-b
          url: jdbc:mysql://master-b:3306/mydb
          priority: 80
          peer: master-a

      slaves:
        - id: slave-a
          url: jdbc:mysql://slave-a:3306/mydb
          master-ref: master-a
          weight: 100
        - id: slave-b
          url: jdbc:mysql://slave-b:3306/mydb
          master-ref: master-b
          weight: 100
```

3. **主从复制支持**

4. **故障转移SQL**
   - 只读模式设置
   - 主从切换命令
   - 健康检测SQL

### 6.7 数据库特定配置示例

# PostgreSQL 配置
spring:
  datasource:
    dbguardian:
      masters:
        - id: pg-master

spring:
  datasource:
    dbguardian:
      masters:
        - id: oracle-master
