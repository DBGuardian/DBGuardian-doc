# DBGuardian - Spring Boot 数据库高可用 Starter

## 1. 项目概述

**项目名称**: DBGuardian
**项目类型**: Spring Boot 数据库高可用自动配置 Starter
**核心价值**: 为 Spring Boot 应用提供**读写分离**、**自动故障转移**、**分布式协调**能力
**目标用户**: 企业级 Spring Boot 应用开发者

**当前状态**: 已完成从原项目的抽离，成为独立的 DBGuardian Spring Boot Starter

## 2. 核心功能

### 2.1 已实现功能

| 功能 | 描述 | 源文件 |
|-----|------|-------|
| **读写分离** | 自动路由读/写操作到不同数据源 | `DbGuardianDataSourceAspect.java` |
| **连接池** | HikariCP 高性能连接池 | `DbGuardianAutoConfiguration.java` |
| **故障转移** | 主库故障时自动将从库升为主库 | `FailoverController.java` |
| **健康检查** | 定期检测主从库健康状态 | `DataSourceHealthChecker.java` |
| **分布式协调** | Redis 实现多实例状态同步 | `DatasourceCoordinationService.java` |
| **状态广播** | 故障转移后通知其他实例 | `DatasourceStatusListener.java` |
| **降级启动** | 允许从库故障时降级运行 | `DbGuardianAutoConfiguration.java` |
| **原主库恢复** | 原主库恢复后作为从库追赶数据 | `FailoverController.java` |
| **负载均衡** | 支持多从库负载均衡 | `loadbalance/*.java` |
| **多主多从** | 支持多主多从集群架构 | `ClusterDataSourceProperties.java` |

### 2.2 支持的集群架构

#### 2.2.1 基础架构模式

| 模式 | 架构类型 | 读能力 | 写能力 | 适用场景 |
|-----|---------|-------|-------|---------|
| **M1S1** | 单主单从 | 1x | 1x | 开发/测试/小规模生产 |
| **M1Sn** | 单主多从 | nx | 1x | 读密集型应用 |
| **MnSn** | 多主多从 | nx | mx | 高可用读写 |
| **MnS0** | 多主无从 | 0x | mx | 分库写入 |
| **M2S2** | 双主双从 | 2x | 2x | 对称高可用 |

#### 2.2.2 高级架构模式

| 模式 | 架构类型 | 读能力 | 写能力 | 适用场景 |
|-----|---------|-------|-------|---------|
| **Cascade** | 级联复制 | nx | 1x | 减轻主库复制压力 |
| **Ring** | 环形复制 | nx | mx | 多数据中心部署 |
| **Star** | 星型复制 | nx | 1x | 大规模从库部署 |
| **Cluster** | 集群模式 | nx | nx | 强一致性高可用 |
| **Sharding** | 分片集群 | nx | nx | 超大规模数据 |
| **Active-Active** | 双活模式 | nx | 2x | 多活数据中心 |

#### 2.2.3 架构说明

**级联复制 (Cascade)**
```
主库 ──> 从库1 ──> 从库2 ──> 从库3
```
- 主库只负责向一级从库复制
- 减轻主库网络压力
- 适合大规模从库部署

**环形复制 (Ring)**
```
主库1 ──> 主库2 ──> 主库3 ──> 主库1
```
- 多主库首尾相连
- 数据中心间复制
- 任一节点故障可继续工作

**星型复制 (Star)**
```
         主库
       /  |  \
    从1  从2  从3 ...
```
- 一个主库，多个直接下游从库
- 结构简单，管理方便
- 适合中等规模部署

**集群模式 (MySQL Group Replication / Galera)**
```
┌─────────────────────────────────┐
│        Group Replication         │
│  节点1 ←→ 节点2 ←→ 节点3 ←→ 节点1 │
└─────────────────────────────────┘
```
- 多节点共识复制
- 自动故障转移
- 强一致性保证

**分片集群 (Sharding)**
```
分片1 (M1S1) ─┐
分片2 (M1S1) ──┼── 应用层
分片3 (M1S1) ──┘
```
- 按业务键分片
- 每个分片独立主从
- 水平扩展写入能力

**双活模式 (Active-Active)**
```
数据中心A ←──双向复制──→ 数据中心B
  主库1                 主库2
```
- 两地都可读写
- 需要解决冲突
- 适合灾备场景

### 2.3 负载均衡策略

| 策略 | 说明 |
|-----|------|
| `weighted-round-robin` | 加权轮询，按权重比例分配请求 |
| `weighted-random` | 加权随机，按权重比例随机选择 |
| `least-connections` | 最少连接，选择活跃连接数最少的后端 |
| `consistent-hash` | 一致性哈希，相同分片键路由到同一节点 |

### 4.3 配置属性设计

```yaml
spring:
  datasource:
    dbguardian:
      # 主库配置
      master:
        url: jdbc:mysql://localhost:3306/test
        username: root
        password: password
        hikari:
          maximum-pool-size: 20
          minimum-idle: 5
          connection-timeout: 30000
          idle-timeout: 600000
          max-lifetime: 1800000
      # 从库配置
      slave:
        url: jdbc:mysql://localhost:3307/test
        username: root
        password: password
        hikari:
          maximum-pool-size: 20
          minimum-idle: 5
      # 故障转移配置
      failover:
        enabled: true
        check-interval: 30s        # 健康检查间隔
        auto-promote-slave: true    # 自动升主
      # 分布式协调配置
      coordination:
        redis:
          enabled: true
          channel: dbguardian:status:channel
          lock-key: dbguardian:failover:lock
      # 高级配置
      advanced:
        allow-degraded-startup: true  # 允许降级启动
```

## 5. 读写分离策略

### 5.1 自动路由规则

| 方法前缀 | 目标数据源 | 说明 |
|---------|-----------|------|
| `select*` | 从库 | 查询操作 |
| `get*` | 从库 | 获取操作 |
| `query*` | 从库 | 查询操作 |
| `find*` | 从库 | 查找操作 |
| `count*` | 从库 | 计数操作 |
| `list*` | 从库 | 列表操作 |
| `page*` | 从库 | 分页操作 |
| `search*` | 从库 | 搜索操作 |
| 其他 | 主库 | 写操作 |

### 5.2 故障转移时的行为

当检测到主库故障：
1. 获取分布式锁（防止多实例同时操作）
2. 将从库提升为主库
3. 广播状态变更
4. 所有操作切换到新主库
5. 释放分布式锁

## 6. 使用示例

### 6.1 添加依赖

```xml
<dependency>
    <groupId>io.dbguardian</groupId>
    <artifactId>dbguardian-spring-boot-starter</artifactId>
    <version>1.0.0</version>
</dependency>
```

### 6.2 配置文件

```yaml
spring:
  datasource:
    dbguardian:
      master:
        url: jdbc:mysql://master-host:3306/mydb
        username: root
        password: secret
      slave:
        url: jdbc:mysql://slave-host:3306/mydb
        username: root
        password: secret
      failover:
        enabled: true
```

### 6.3 自动使用

```java
@Service
public class UserService {

    // 读操作 - 自动路由到从库
    public User getUser(Long id) {
        return userMapper.selectById(id);  // select → 从库
    }

    // 写操作 - 自动路由到主库
    public void createUser(User user) {
        userMapper.insert(user);  // insert → 主库
    }

    // 故障转移时自动切换
    public void processOrder(Order order) {
        // 事务内自动使用主库
        orderMapper.insert(order);
    }
}
```
