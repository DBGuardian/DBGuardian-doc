# DBGuardian 集群架构设计方案

## 1. 概述

本文档描述 DBGuardian 支持的多种数据库集群拓扑结构，包括单主单从、单主多从、多主多从、纯主库集群等架构的设计方案。

### 1.1 基础架构模式

| 模式 | 架构类型 | 读能力 | 写能力 | 适用场景 |
|-----|---------|-------|-------|---------|
| **M1S1** | 单主单从 | 1x | 1x | 开发/测试/小规模生产 |
| **M1Sn** | 单主多从 | nx | 1x | 读密集型应用 |
| **MnSn** | 多主多从 | nx | mx | 高可用读写 |
| **MnS0** | 多主无从 | 0x | mx | 分库写入 |
| **M2S2** | 双主双从 | 2x | 2x | 对称高可用 |

### 1.2 高级架构模式

| 模式 | 架构类型 | 读能力 | 写能力 | 适用场景 |
|-----|---------|-------|-------|---------|
| **Cascade** | 级联复制 | nx | 1x | 减轻主库复制压力 |
| **Ring** | 环形复制 | nx | mx | 多数据中心部署 |
| **Star** | 星型复制 | nx | 1x | 大规模从库部署 |
| **GR** | Group Replication | nx | nx | 强一致性高可用 |
| **Galera** | Galera集群 | nx | nx | 同步多主复制 |
| **Sharding** | 分片集群 | nx | nx | 超大规模数据 |

### 1.3 架构对比

| 架构 | 一致性 | 可用性 | 扩展性 | 复杂度 | 延迟 |
|-----|-------|-------|-------|-------|-----|
| 主从复制 | 异步 | 高 | 读扩展 | 低 | 低 |
| 级联复制 | 异步 | 高 | 读扩展 | 中 | 较高 |
| 环形复制 | 异步 | 中 | 中 | 高 | 高 |
| Group Replication | 最终一致 | 很高 | 中 | 高 | 低 |
| Galera | 同步 | 很高 | 中 | 高 | 中 |
| 分片集群 | 最终一致 | 高 | 高 | 很高 | 低 |

## 4. 配置模型

### 4.1 统一配置结构

```yaml
spring:
  datasource:
    dbguardian:
      enabled: true
      
      # 集群模式: single-master | single-master-multi-slave | multi-master | multi-master-no-slave
      cluster-mode: single-master-multi-slave
      
      # 主库配置（支持多个）
      masters:
        - id: master-1
          url: jdbc:mysql://master1:3306/mydb
          username: root
          password: secret
          priority: 100  # 优先级，数值越高越优先
          hikari:
            maximum-pool-size: 20
            minimum-idle: 5
            
        - id: master-2
          url: jdbc:mysql://master2:3306/mydb
          username: root
          password: secret
          priority: 80
          hikari:
            maximum-pool-size: 20
            minimum-idle: 5
      
      # 从库配置（支持多个）
      slaves:
        - id: slave-1
          url: jdbc:mysql://slave1:3306/mydb
          weight: 100  # 负载权重
          hikari:
            maximum-pool-size: 20
            
        - id: slave-2
          url: jdbc:mysql://slave2:3306/mydb
          weight: 80
          hikari:
            maximum-pool-size: 20
            
        - id: slave-3
          url: jdbc:mysql://slave3:3306/mydb
          weight: 60
          hikari:
            maximum-pool-size: 20
      
      # 负载均衡策略
      load-balance:
        # 读操作负载均衡策略: random | round-robin | weight | least-connections
        read-strategy: weight
        # 写操作路由策略: hash | random | primary-only
        write-strategy: hash
        
      # 故障转移配置
      failover:
        enabled: true
        check-interval: 30s
        max-retries: 3
        # 故障转移时是否自动将原主库降为从库
        demote-on-failover: true
```

## 4. 核心组件设计

### 4.1 数据源注册中心

```java
/**
 * 数据源注册中心
 * 统一管理所有主库和从库的生命周期
 */
public class DataSourceRegistry {
    
    // 主库Map: id -> DataSourceWrapper
    private final Map<String, DataSourceWrapper> masters = new ConcurrentHashMap<>();
    
    // 从库Map: id -> DataSourceWrapper
    private final Map<String, DataSourceWrapper> slaves = new ConcurrentHashMap<>();
    
    // 可用主库（心跳检测后更新）
    private final AtomicReference<Set<String>> availableMasters = new AtomicReference<>(new HashSet<>());
    
    // 可用从库
    private final AtomicReference<Set<String>> availableSlaves = new AtomicReference<>(new HashSet<>());
    
    /**
     * 注册主库
     */
    public void registerMaster(String id, DataSourceWrapper ds) {
        masters.put(id, ds);
    }
    
    /**
     * 注册从库
     */
    public void registerSlave(String id, DataSourceWrapper ds) {
        slaves.put(id, ds);
    }
    
    /**
     * 获取所有可用主库
     */
    public List<DataSourceWrapper> getAvailableMasters() {
        return availableMasters.get().stream()
            .map(masters::get)
            .filter(Objects::nonNull)
            .collect(Collectors.toList());
    }
    
    /**
     * 获取所有可用从库
     */
    public List<DataSourceWrapper> getAvailableSlaves() {
        return availableSlaves.get().stream()
            .map(slaves::get)
            .filter(Objects::nonNull)
            .collect(Collectors.toList());
    }
    
    /**
     * 更新可用状态
     */
    public void updateAvailability(String id, boolean isMaster, boolean available) {
        Set<String> set = isMaster ? availableMasters.get() : availableSlaves.get();
        Set<String> newSet = new HashSet<>(set);
        if (available) {
            newSet.add(id);
        } else {
            newSet.remove(id);
        }
        if (isMaster) {
            availableMasters.set(newSet);
        } else {
            availableSlaves.set(newSet);
        }
    }
}
```

### 4.2 负载均衡策略接口

```java
/**
 * 读操作负载均衡策略
 */
public interface ReadLoadBalanceStrategy {
    
    /**
     * 选择一个从库
     * @param availableSlaves 可用的从库列表
     * @param routingContext 路由上下文（包含分片键等）
     * @return 选中的从库
     */
    DataSourceWrapper select(List<DataSourceWrapper> availableSlaves, RoutingContext routingContext);
    
    /**
     * 策略名称
     */
    String name();
}
```

#### 4.2.1 轮询策略

```java
/**
 * 加权轮询负载均衡
 */
@Component
public class WeightedRoundRobinStrategy implements ReadLoadBalanceStrategy {
    
    private final AtomicInteger counter = new AtomicInteger(0);
    
    @Override
    public DataSourceWrapper select(List<DataSourceWrapper> slaves, RoutingContext context) {
        if (slaves.isEmpty()) return null;
        
        // 计算总权重
        int totalWeight = slaves.stream()
            .mapToInt(DataSourceWrapper::getWeight)
            .sum();
        
        // 轮询选择
        int index = counter.getAndIncrement() % slaves.size();
        
        // 按权重调整（高权重节点被选中概率更高）
        int selected = index;
        int currentWeight = 0;
        for (int i = 0; i < slaves.size(); i++) {
            currentWeight += slaves.get(i).getWeight();
            if (currentWeight > (index * totalWeight / slaves.size())) {
                selected = i;
                break;
            }
        }
        return slaves.get(selected);
    }
    
    @Override
    public String name() {
        return "weighted-round-robin";
    }
}
```

#### 4.2.2 随机策略

```java
/**
 * 加权随机负载均衡
 */
@Component
public class WeightedRandomStrategy implements ReadLoadBalanceStrategy {
    
    private final Random random = new Random();
    
    @Override
    public DataSourceWrapper select(List<DataSourceWrapper> slaves, RoutingContext context) {
        if (slaves.isEmpty()) return null;
        if (slaves.size() == 1) return slaves.get(0);
        
        int totalWeight = slaves.stream()
            .mapToInt(DataSourceWrapper::getWeight)
            .sum();
        
        int rand = random.nextInt(totalWeight);
        int cumulative = 0;
        for (DataSourceWrapper slave : slaves) {
            cumulative += slave.getWeight();
            if (rand < cumulative) {
                return slave;
            }
        }
        return slaves.get(slaves.size() - 1);
    }
    
    @Override
    public String name() {
        return "weighted-random";
    }
}
```

#### 4.2.3 最少连接策略

```java
/**
 * 最少连接负载均衡
 */
@Component
public class LeastConnectionsStrategy implements ReadLoadBalanceStrategy {
    
    @Override
    public DataSourceWrapper select(List<DataSourceWrapper> slaves, RoutingContext context) {
        if (slaves.isEmpty()) return null;
        
        return slaves.stream()
            .min(Comparator.comparingInt(DataSourceWrapper::getActiveConnections))
            .orElse(slaves.get(0));
    }
    
    @Override
    public String name() {
        return "least-connections";
    }
}
```

```java
 * 一致性哈希负载均衡
 */
@Component
public class ConsistentHashStrategy implements ReadLoadBalanceStrategy {
    
    private final ConcurrentHashMap<Long, DataSourceWrapper> ring = new ConcurrentHashMap<>();
    private final int virtualNodes = 100;
    
    @Override
    public DataSourceWrapper select(List<DataSourceWrapper> slaves, RoutingContext context) {
        if (slaves.isEmpty()) return null;
        
        // 使用分片键计算哈希
        String shardKey = context.getShardKey();
        if (shardKey == null) {
            shardKey = String.valueOf(random.nextInt());
        }
        
        long hash = hash(shardKey);
        
        // 顺时针查找节点
        Long key = ring.ceilingKey(hash);
        if (key == null) {
            key = ring.firstKey();
        }
        return ring.get(key);
    }
    
    private long hash(String key) {
        return key.hashCode() & 0xffffffffL;
    }
    
    /**
     * 重新构建哈希环
     */
    public void rebuildRing(List<DataSourceWrapper> slaves) {
        ring.clear();
        for (DataSourceWrapper slave : slaves) {
            for (int i = 0; i < virtualNodes; i++) {
                long hash = hash(slave.getId() + "-" + i);
                ring.put(hash, slave);
            }
        }
    }
    
    @Override
    public String name() {
        return "consistent-hash";
    }
}
```

```java
/**
 * 多主多从路由数据源
 */
public class MultiMasterSlaveRoutingDataSource extends AbstractRoutingDataSource {
    
    @Autowired
    private DataSourceRegistry registry;
    
    @Autowired
    private Map<String, ReadLoadBalanceStrategy> readStrategies;
    
    @Value("${spring.datasource.dbguardian.load-balance.read-strategy:weight}")
    private String readStrategyName;
    
    @Override
    protected Object determineCurrentLookupKey() {
        RoutingContext ctx = RoutingContext.get();
        
        // 显式标记主库
        if (ctx != null && ctx.isForceMaster()) {
            return DataSourceType.MASTER;
        }
        
        // 写操作始终路由到主库
        if (ctx != null && ctx.getOperationType() == OperationType.WRITE) {
            return selectMaster();
        }
        
        // 读操作负载均衡到从库
        if (ctx != null && ctx.getOperationType() == OperationType.READ) {
            return selectSlave();
        }
        
        // 默认使用主库
        return DataSourceType.MASTER;
    }
    
    /**
     * 选择主库
     */
    private DataSourceType selectMaster() {
        List<DataSourceWrapper> available = registry.getAvailableMasters();
        if (available.isEmpty()) {
            throw new NoAvailableDataSourceException("无可用主库");
        }
        
        // 按优先级选择
        return available.stream()
            .max(Comparator.comparingInt(DataSourceWrapper::getPriority))
            .map(w -> DataSourceType.MASTER)
            .orElse(DataSourceType.MASTER);
    }
    
    /**
     * 选择从库
     */
    private DataSourceType selectSlave() {
        List<DataSourceWrapper> available = registry.getAvailableSlaves();
        if (available.isEmpty()) {
            // 降级到主库
            log.warn("无可用从库，降级到主库");
            return DataSourceType.MASTER;
        }
        
        // 使用配置的负载均衡策略
        ReadLoadBalanceStrategy strategy = readStrategies.get(readStrategyName);
        if (strategy == null) {
            strategy = readStrategies.values().iterator().next();
        }
        
        DataSourceWrapper selected = strategy.select(available, RoutingContext.get());
        return selected != null ? DataSourceType.SLAVE : DataSourceType.MASTER;
    }
}
```

### 4.5 故障转移控制器

```java
/**
 * 故障转移控制器
 * 负责检测故障并执行故障转移
 */
public class FailoverController {
    
    @Autowired
    private DataSourceRegistry registry;
    
    @Autowired
    private DatasourceCoordinationService coordinationService;
    
    /**
     * 处理主库故障
     */
    public synchronized void handleMasterFailure(String failedMasterId) {
        log.warn("检测到主库 {} 故障，开始故障转移", failedMasterId);
        
        // 1. 获取分布式锁
        if (!coordinationService.tryAcquireFailoverLock()) {
            log.info("其他实例正在处理故障转移");
            return;
        }
        
        try {
            // 2. 查找备用主库
            List<DataSourceWrapper> available = registry.getAvailableMasters();
            available = available.stream()
                .filter(w -> !w.getId().equals(failedMasterId))
                .collect(Collectors.toList());
            
            if (available.isEmpty()) {
                log.error("没有备用主库，故障转移失败");
                return;
            }
            
            // 3. 选择优先级最高的主库
            DataSourceWrapper newMaster = available.stream()
                .max(Comparator.comparingInt(DataSourceWrapper::getPriority))
                .orElse(available.get(0));
            
            // 4. 通知其他从库切换主库
            notifySlavesNewMaster(newMaster);
            
            // 5. 广播状态变更
            coordinationService.broadcastFailoverEvent(failedMasterId, newMaster.getId());
            
            log.info("故障转移完成: {} -> {}", failedMasterId, newMaster.getId());
            
        } finally {
            coordinationService.releaseFailoverLock();
        }
    }
    
    /**
     * 处理从库故障
     */
    public void handleSlaveFailure(String failedSlaveId) {
        log.warn("检测到从库 {} 故障", failedSlaveId);
        registry.updateAvailability(failedSlaveId, false, false);
        coordinationService.broadcastSlaveStatus(failedSlaveId, false);
    }
    
    /**
     * 通知从库切换主库
     */
    private void notifySlavesNewMaster(DataSourceWrapper newMaster) {
        for (DataSourceWrapper slave : registry.getAvailableSlaves()) {
            try {
                reconfigureSlave(slave, newMaster);
            } catch (Exception e) {
                log.error("重新配置从库 {} 失败: {}", slave.getId(), e.getMessage());
            }
        }
    }
    
    /**
     * 重新配置从库连接到新主库
     */
    private void reconfigureSlave(DataSourceWrapper slave, DataSourceWrapper newMaster) {
        try (Connection conn = slave.getDataSource().getConnection();
             Statement stmt = conn.createStatement()) {
            
            stmt.execute("STOP SLAVE");
            stmt.execute("RESET SLAVE ALL");
            
            String sql = String.format(
                "CHANGE MASTER TO MASTER_HOST='%s', MASTER_PORT=%d, MASTER_USER='%s', MASTER_PASSWORD='%s', MASTER_AUTO_POSITION=1",
                extractHost(newMaster.getUrl()),
                extractPort(newMaster.getUrl()),
                replicationUser,
                replicationPassword
            );
            stmt.execute(sql);
            stmt.execute("START SLAVE");
            
            log.info("从库 {} 已重新配置连接到新主库 {}", slave.getId(), newMaster.getId());
            
        } catch (SQLException e) {
            log.error("重新配置从库失败: {}", e.getMessage());
        }
    }
}
```

### 4.6 心跳检测器

```java
/**
 * 数据源心跳检测器
 */
@Component
public class DataSourceHealthChecker {
    
    private final ScheduledExecutorService executor = Executors.newScheduledThreadPool(2);
    
    @Autowired
    private DataSourceRegistry registry;
    
    @Autowired
    private FailoverController failoverController;
    
    /**
     * 启动所有数据源的心跳检测
     */
    @PostConstruct
    public void startHealthChecks() {
        // 每10秒检测一次主库
        executor.scheduleAtFixedRate(
            this::checkMasters,
            0, 10, TimeUnit.SECONDS
        );
        
        // 每5秒检测一次从库
        executor.scheduleAtFixedRate(
            this::checkSlaves,
            0, 5, TimeUnit.SECONDS
        );
    }
    
    /**
     * 检测主库健康状态
     */
    private void checkMasters() {
        for (DataSourceWrapper master : registry.getAllMasters()) {
            boolean healthy = isHealthy(master);
            boolean wasAvailable = registry.isAvailable(master.getId(), true);
            
            registry.updateAvailability(master.getId(), true, healthy);
            
            if (wasAvailable && !healthy) {
                log.error("主库 {} 不可用", master.getId());
                failoverController.handleMasterFailure(master.getId());
            } else if (!wasAvailable && healthy) {
                log.info("主库 {} 恢复", master.getId());
            }
        }
    }
    
    /**
     * 检测从库健康状态
     */
    private void checkSlaves() {
        for (DataSourceWrapper slave : registry.getAllSlaves()) {
            boolean healthy = isHealthy(slave);
            boolean wasAvailable = registry.isAvailable(slave.getId(), false);
            
            registry.updateAvailability(slave.getId(), false, healthy);
            
            if (wasAvailable && !healthy) {
                log.warn("从库 {} 不可用，已从负载均衡池移除", slave.getId());
            }
        }
    }
    
    /**
     * 健康检测
     */
    private boolean isHealthy(DataSourceWrapper ds) {
        try (Connection conn = ds.getDataSource().getConnection()) {
            return conn.isValid(5);
        } catch (SQLException e) {
            return false;
        }
    }
}
```

## 5. 集群模式详细设计

### 5.1 M1Sn - 单主多从

**特点：**
- 写入始终路由到唯一主库
- 读取通过负载均衡分发到多个从库
- 主库故障时从库自动升主

**配置示例：**

```yaml
spring:
  datasource:
    dbguardian:
      cluster-mode: single-master-multi-slave
      
      masters:
        - id: master-1
          url: jdbc:mysql://master:3306/mydb
          priority: 100
          
      slaves:
        - id: slave-1
          url: jdbc:mysql://slave1:3306/mydb
          weight: 100
        - id: slave-2
          url: jdbc:mysql://slave2:3306/mydb
          weight: 100
        - id: slave-3
          url: jdbc:mysql://slave3:3306/mydb
          weight: 80
```

**读流量分配（按权重）：**
```
总权重 = 100 + 100 + 80 = 280

请求1 -> slave-1 (0-100)
请求2 -> slave-2 (100-200)
请求3 -> slave-3 (200-280)
请求4 -> slave-1 (0-100)
...
```

### 5.2 MnSn - 多主多从

**特点：**
- 写入根据分片键路由到对应主库
- 每个主库可配置多个从库
- 主库之间互为备份

**配置示例：**

```yaml
spring:
  datasource:
    dbguardian:
      cluster-mode: multi-master-multi-slave
      sharding-column: user_id  # 分片键
      
      masters:
        - id: master-0
          url: jdbc:mysql://master0:3306/mydb_0
          priority: 100
        - id: master-1
          url: jdbc:mysql://master1:3306/mydb_1
          priority: 100
          
      slaves:
        - id: slave-0-1
          url: jdbc:mysql://slave0:3306/mydb_0
          master-ref: master-0
          weight: 100
        - id: slave-0-2
          url: jdbc:mysql://slave1:3306/mydb_0
          master-ref: master-0
          weight: 80
        - id: slave-1-1
          url: jdbc:mysql://slave2:3306/mydb_1
          master-ref: master-1
          weight: 100
```

### 5.3 MnS0 - 多主无从

**特点：**
- 所有节点都是主库，可写入
- 无从库分担读压力
- 适用于分库写入场景

**配置示例：**

```yaml
spring:
  datasource:
    dbguardian:
      cluster-mode: multi-master-no-slave
      sharding-column: order_id
      
      masters:
        - id: shard-0
          url: jdbc:mysql://db0:3306/mydb
          priority: 100
        - id: shard-1
          url: jdbc:mysql://db1:3306/mydb
          priority: 100
        - id: shard-2
          url: jdbc:mysql://db2:3306/mydb
          priority: 100
        - id: shard-3
          url: jdbc:mysql://db3:3306/mydb
          priority: 100
```

### 5.4 M2S2 - 双主双从

**特点：**
- 两个主库互为备份
- 两个从库分别跟随对应主库
- 主库故障时自动切换

**配置示例：**

```yaml
spring:
  datasource:
    dbguardian:
      cluster-mode: dual-master-dual-slave
      
      masters:
        - id: master-a
          url: jdbc:mysql://master-a:3306/mydb
          priority: 100
          peer: master-b  # 故障转移目标
        - id: master-b
          url: jdbc:mysql://master-b:3306/mydb
          priority: 80
          peer: master-a
          
      slaves:
        - id: slave-a
          url: jdbc:mysql://slave-a:3306/mydb
          master-ref: master-a
          weight: 100
        - id: slave-b
          url: jdbc:mysql://slave-b:3306/mydb
          master-ref: master-b
          weight: 100
```

## 7. 故障转移流程

### 7.1 主库故障转移

```
1. 心跳检测发现 Master-A 不可用
           │
           ▼
2. 获取分布式锁 (防止多实例同时操作)
           │
           ▼ (成功获取锁)
3. 查询所有主库优先级
           │
           ▼
4. 选择优先级最高的主库 (Master-B, priority=100)
           │
           ▼
5. 向 Redis 广播: MASTER_FAILOVER {old: A, new: B}
           │
           ▼
6. 通知所有从库切换主库
   - Slave-A' 重新配置指向 Master-B
   - Slave-B' 重新配置指向 Master-B
           │
           ▼
7. 更新路由表
           │
           ▼
8. 释放分布式锁
           │
           ▼
9. 其他实例收到广播，同步状态
```

### 7.2 从库故障转移

```
1. 心跳检测发现 Slave-A' 不可用
           │
           ▼
2. 从可用从库列表移除 Slave-A'
           │
           ▼
3. 读请求不再路由到 Slave-A'
           │
           ▼
4. 向 Redis 广播: SLAVE_DOWN {id: A'}
           │
           ▼
5. 其他实例同步可用从库列表
           │
           ▼
6. (可选) 如果配置了从库自动恢复，尝试重新配置
```

## 8. 数据一致性考虑

### 8.1 读写分离延迟

```
主库写入 ──────(复制延迟 0.5-2s)──────> 从库可读

问题: 写入后立即读取可能读到旧数据

解决方案:
1. 强一致性读: 写入后强制读主库
2. 延迟感知路由: 基于复制延迟动态决策
3. 应用层补偿: 写入后短暂时间内读主库
```

### 8.2 分布式事务

```
问题: 多主库场景下的分布式事务

解决方案:
1. 避免跨库事务: 按分片键设计业务
2. 最终一致性: 使用消息队列+补偿机制
3. XA 事务: 使用 Seata 等分布式事务框架
```

### 8.3 故障恢复数据校验

```
主库故障 ──> 从库升主 ──> 原主库恢复

问题: 原主库可能存在未复制的脏数据

解决方案:
1. GTID 比对: 检查复制位点
2. 延迟恢复: 等待一段时间后恢复
3. 手动确认: 管理员确认后执行
```
